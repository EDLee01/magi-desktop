### Layered Memory Foundation

Implemented first usable layered memory:

- User memory: `~/.magi-next/memory.md`
- Project memory: `~/.magi-next/state/project-memory/<project-id>.md`
- Session memory: `~/.magi-next/state/session-memory/<session-id>.md`
- Deterministic keyword retrieval across user/project/session scopes.
- QueryEngine injects matching results as:

```text
[Relevant memory]
- session: ...
- project: ...
- user: ...
```

- Explicit memory writes only:
  - `remember project: ...`
  - `remember session: ...`
  - `remember user: ...`
  - `记住用户记忆：...`
  - `记住项目记忆：...`
  - `记住会话记忆：...`
- Ordinary chat is not inferred into memory.
- Duplicate entries are skipped.
- Same-scope key/value conflicts are skipped and audited.
- CLI supports:
  - `magi memory view [user|project|session] [--session-id <id>]`
  - `magi memory search <query> [--session-id <id>]`
  - `magi memory append <user|project|session> <text> [--session-id <id>]`
- Config supports:
  - `memory.enabled`
  - `memory.autoWrite` with `off` or `explicit`
  - `memory.maxResults`
  - `memory.scopes`

## Current Product Shape

Magi Next is still a TypeScript-first clean-room implementation with a Rust runner sidecar.

Working foundations:

- Agent loop with tool calls and tool-result feedback.
- Provider fallback.
- MCP dynamic tools/resources.
- Hooks and durable audit events.
- Control API CRUD/admin endpoints.
- Real SSE audit stream foundation.
- Active Control API approval/question wait and resolution for running jobs.
- Readline-level TUI with live audit-event output, compact transcript/status formatting, session/model picker affordances, and interactive approval/question controls.
- Context compaction.
- Layered memory retrieval and explicit memory write policy.
- 30 built-in tools.

Known limits:

- Provider streaming now covers OpenAI-compatible and Anthropic Messages-compatible SSE, but richer terminal streaming presentation is still readline-level.
- Control API cancellation now works for running background jobs; synchronous HTTP job cancellation, tool-level process cancellation, and TUI keyboard interrupt semantics still need hardening.
- TUI is still readline-level, not a full-screen transcript UI; richer overlays/file mention pickers/task views are still missing.
- Memory ranking is deterministic keyword scoring only.
- Memory conflict handling audits conflicts but does not resolve them interactively.
- No real sub-agent execution yet.
- Rust runner process isolation and PTY behavior are still basic.
