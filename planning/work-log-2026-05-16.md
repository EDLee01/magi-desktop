Details:

- `Grep` now supports path, glob, type, regex/fixed string, ignore case, context lines, head limit, max matches, line number toggle, output modes.
- `WebFetch` fetches real HTTP(S), extracts readable HTML text, and summarizes through active model route. Non-allowlisted URLs ask for approval.
- `AskUserQuestion` supports 1-4 structured questions, 2-4 options, single/multi-select, resolver validation, TUI resolver, and agent `user_question` events.
- `SendUserMessage` and `Brief` support message, attachments, `normal|proactive`, sink delivery, and agent `user_message` events.
- `Cron*` tools store jobs in `~/.magi-next/state/cron-jobs.json`, parse 5-field cron expressions, support create/update/delete/list, and Control API runs due jobs through the headless provider loop.
- `TodoWrite` replaces a session todo list with validated `{ id, content, status, priority? }` items, persists state under `~/.magi-next/state/todos.json`, returns the current state as a tool result, and records dedicated QueryEngine todo update audit.
- `ToolSearch` searches built-in tool metadata and supports `select:<tool_name>` for full schema output.
- `Config` reads and updates allowlisted Magi Next config keys in `~/.magi-next/config.yaml`, validates writes through the normal config loader, and records QueryEngine config update audit.
- `Skill` lists and loads skills from Magi Next `skillsRoot`, rejects traversal-style names, returns selected skill instructions as tool results, and records QueryEngine skill load audit.
- `LSP` uses TypeScript compiler API for real TS/JS semantics:
  - `goToDefinition`
  - `findReferences`
  - `hover`
  - `documentSymbol`
  - `workspaceSymbol`
  - `goToImplementation`
  - `prepareCallHierarchy`
  - `incomingCalls`
  - `outgoingCalls`

### H4 Hooks Expansion

- Hook event schema now covers the broader clean-room event family from the decomposition:
  - tool events: `pre_tool_use`, `post_tool_use`, `post_tool_use_failure`
  - session/query events: `session_start`, `session_end`, `user_prompt_submit`
  - permission events: `permission_request`, `permission_denied`
  - subagent/task events: `subagent_start`, `subagent_stop`, `task_created`, `task_completed`
  - lifecycle/config/file/other events: `pre_compact`, `post_compact`, `teammate_idle`, `elicitation`, `elicitation_result`, `config_change`, `worktree_create`, `worktree_remove`, `instructions_loaded`, `cwd_changed`, `file_changed`, `notification`, `setup`, `stop`, `stop_failure`
- Hook matcher supports existing `ToolName(selector)` plus context equality such as `agentType:worker` and broad glob matching against event context.
- QueryEngine now emits and audits `user_prompt_submit`, `permission_request`, `permission_denied`, provider fallback `notification`, and Config `config_change` hooks.
- CLI and Control API agent task flows now trigger `task_created`, `subagent_start`, `task_completed`, and `subagent_stop` hooks in addition to existing notification/stop hooks.
