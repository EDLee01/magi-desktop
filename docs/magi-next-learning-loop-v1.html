Magi vNext proposal

Learning Loop v1

下一版本的目标不是继续堆工具，而是让 Magi 从“能执行任务的 agent” 进化成“能从任务中学习、复用经验、更新技能的 agent”。

Target: next minor version Theme: self-improving harness Source inspiration: Hermes Agent Scope: local-first, reviewable learning Status: v1 core implemented

Product Thesis

核心判断

Hermes 最值得吸收的不是多平台 gateway，也不是庞大的 provider 列表， 而是闭环学习能力：任务前检索经验，任务中应用经验，任务后把稳定经验沉淀成 memory 或 skill。

Implementation Status

Shipped

Pre-task Recall

QueryEngine now injects fenced Memory, Skill, and prior session snippets before provider calls.

Shipped

LearningDraft

Learning Flow

Learning Loop v1 的主路径保持可解释、可审核、可回滚。Magi 不应该悄悄改写自己的长期知识， 而是先生成草稿，再由用户批准。

1. Pre-task recall 新任务开始前，检索 memory、skill、session history，注入最相关上下文。
2. Execute 按现有 agent loop 工作，保留 tool trace、失败模式和用户反馈。
3. Reflect 复杂任务结束后，判断是否出现稳定事实、可复用流程或工具教训。
4. Draft 生成 Learning Draft，分类为 memory、skill patch、new skill 或 do-not-save。
5. Approve 用户审核后写入 memdir 或 skills；拒绝的草稿保留 audit。

Features To Bring In

Priority Feature What We Take From Hermes Magi Version Acceptance Criteria
P0 SessionSearch FTS-style cross-session recall with browse, search, and anchored scroll. Deferred tool backed by Magi session SQLite. User says “上次那个问题”, Magi can find the prior session without asking for a recap.
P0 LearningDraft Post-task reflection and reviewable learning proposals. Unify memory draft plus skill draft under one review command. Complex task produces at most 3 high-signal drafts with reason, target, and confidence.
P0 SkillManage Agent-managed procedural memory: create, patch, write support files. Strict path-limited tool for creating and patching skills after approval. Agent can create a reusable skill from a solved workflow; user can approve or reject.
P1 Tool Loop Guardrails Detect repeated exact failures and no-progress loops. Per-turn guardrail in QueryEngine before/after tool calls. Same tool plus same args failing repeatedly triggers a strategy-change warning, then a block.
P1 Tool Result Budget Aggregate per-turn result budget, not only per-tool truncation. Spill oversized combined tool outputs to files with short previews. A turn with many medium outputs cannot blow up prompt tokens.
P1 @ Context References User-controlled context injection such as @file, @diff, @staged. Small preprocessor for @file:path:line-line, @diff, @staged. User can precisely attach context without waiting for the agent to discover files.
P2 Context Fencing Fence memory/context so recalled text cannot masquerade as instructions. Wrap memory and session recall in clear non-user-input blocks. Injected memory is treated as background data, not new user commands.
P2 Model Capability Checks Filter models by tool support and usable context window. Lightweight capability metadata in router and doctor. Non-tool models are not selected for coding agent mode without warning.

Version Scope

Must ship

Learning Loop Core

SessionSearch, LearningDraft, SkillManage, and pre-task recall. This is the minimum that makes the release feel meaningfully smarter.

Should ship

Harness Protection

Tool loop guardrails and aggregate output budget. These reduce wasted tokens and prevent the agent from repeating low-value actions.

Can wait

Platform Expansion

Telegram, Slack, Discord, Modal, Daytona, and external memory providers stay out of this release. Mobile web panel is enough for now.

Proposed Data Shape

One reviewable draft model can cover both memory and skills. It keeps the learning layer auditable instead of silently mutating long-term state.

Implementation Checklist

Phase 1

Recall Before Work

  • SessionSearch tool: search sessions by query with snippets and message windows.
  • Pre-task retrieval: combine memory, skills, and session hits into a compact context block.
  • Context fencing: label recall as background, not user instruction.
Phase 2

Learn After Work

  • Reflection pass: trigger after complex tasks or explicit user feedback.
  • LearningDraft command: list, show, apply, reject.
  • SkillManage: create and patch skills through approval-gated writes.
Phase 3

Protect The Loop

  • Tool loop guardrail: repeated failure warning and block thresholds.
  • Output budget: per-turn aggregate budget with persisted full output.
  • Audit trail: record why learning drafts were proposed, applied, or rejected.
Phase 4

Polish

  • Doctor checks: validate memory roots, skills root, and session FTS index.
  • TUI review: readable learning draft approval surface.
  • Docs: update README and add examples for learning workflow.

Non-goals

No silent self-modification

Magi can propose learning, but user approval is required before durable memory or skill writes.

No full Hermes platform clone

Messaging gateways, cloud terminals, and external memory providers are outside this version.

No tool-count race

New learning tools should be deferred or searched, not dumped into the core schema by default.

Release Definition

Done means Magi can demonstrate learning end to end.

Run a complex task, produce a reviewed LearningDraft, apply it as memory or a skill, then start a new task where Magi retrieves and uses that learned context without user prompting.

Magi vNext Learning Loop v1 - local-first, reviewable, and focused on durable agent improvement.
